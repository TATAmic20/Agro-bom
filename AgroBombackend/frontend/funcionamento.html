Como organizei a parte visual do sistema

Esta página explica, de forma simples, como as telas foram organizadas e como o front-end ajuda o usuário a navegar pelo AGROBOM.


🧩 Organização geral

O front-end foi dividido em páginas separadas. Cada página representa uma parte do sistema, como clientes, produtos, pedidos, estoque e relatórios. Essa divisão deixou o projeto mais organizado e mais fácil de apresentar.



Estrutura das páginas

Cada módulo do sistema possui sua própria página HTML, facilitando a navegação e a organização do projeto.

Menu lateral

O menu lateral conecta todas as telas e permite que o usuário acesse rapidamente cada área do sistema.

Tabelas

As tabelas organizam as informações principais, como clientes cadastrados, produtos, pedidos e estoque.

Modais

Os modais são janelas usadas para cadastrar ou editar dados sem precisar sair da tela atual.

Interações

O JavaScript foi usado para controlar botões, abrir janelas, atualizar tabelas e simular ações do sistema.

Conexão com o projeto

O front-end funciona como a parte visual que representa as informações que seriam tratadas pelo back-end e banco.



🔄 Fluxo das telas

A navegação foi pensada para seguir uma sequência simples: o usuário entra no sistema, visualiza o painel principal, acessa os módulos e consulta as informações necessárias.
